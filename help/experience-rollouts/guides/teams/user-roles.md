# 使用者角色 {#user-roles}

體驗轉出提供五個角色。 每個角色都是針對一組特定責任而設計。 依預設，角色是互斥的 — 具有管理員角色的成員不會自動擁有產品發行擁有者許可權。 當需要更廣泛的存取權時，指派多個角色給成員。

## 可用的角色 {#roles}

| 角色 | 責任 |
|---|---|
| **管理員** | 將應用程式上線到主控台、管理團隊成員，以及核准或拒絕存取請求。 在團隊可以開始使用功能標幟之前需要。 |
| **產品發行擁有者** | 為其應用程式建立和更新功能標幟和功能群組。 可將受眾連結至功能標幟，使功能可供外部使用者使用。 |
| **開發人員** | 在沙箱化內容中運作，以測試功能標幟後的功能，而不會影響其他使用者。 可在Stage或Production中將功能標幟公開給私人對象（例如特定使用者ID或電子郵件地址）。 |
| **功能管理員** | 建立和管理跨團隊功能群組。 在協調不同團隊擁有的應用程式旗標時，請使用此角色。 |
| **發行管理員** | 管理多團隊、多應用程式版本，並更新定義誰會收到版本的對象規則。 |

## 選擇正確的角色 {#choosing}

使用下列指南來指派角色：

* **正在新增或更新應用程式的功能標幟** → **產品發行擁有者**
* **在不影響其他人的情況下私密測試功能** → **開發人員**
* **管理跨團隊功能群組** → **功能管理員**
* **跨多個團隊和應用程式管理版本** → **版本管理員**
* **上線應用程式及管理團隊成員資格** → **管理員**

>[!NOTE]
>
>一個成員可以擔任多個角色。 例如，同時測試功能和管理團隊應用程式的工程師應同時具有&#x200B;**開發人員**&#x200B;和&#x200B;**管理員**&#x200B;角色。

## 權限錯誤 {#permissions-error}

如果成員在嘗試更新功能或群組時遇到「許可權不足」錯誤，他們需要&#x200B;**產品發行擁有者**&#x200B;角色。 請聯絡您的團隊管理員以新增此角色。
