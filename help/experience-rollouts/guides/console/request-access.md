# 要求存取權 {#request-access}

體驗轉出的存取權在團隊層級管理。 若要使用主控台，您需要將自己新增到具有適當角色的團隊中。

## 先決條件 {#prerequisites}

在請求存取權之前，請確認下列事項：

* 您知道要加入哪個團隊。 請洽詢您的產品或工程負責人，以識別您的團隊及其管理員。
* 您知道您需要存取哪個環境 — **階段**、**生產**&#x200B;或兩者。

## 要求存取權 {#request-steps}

1. 導覽至您組織的存取管理系統。
2. 搜尋&#x200B;**體驗轉出**，並為您的環境（預備或生產）選取適當的應用程式。
3. 搜尋您的團隊名稱，並選取您要求的角色。
4. 提交請求。

請求提交後，您將會收到確認。 您可以在&#x200B;**擱置中**&#x200B;要求的存取管理系統中監視要求的狀態。

>[!NOTE]
>
>在您團隊中具有&#x200B;**管理員**&#x200B;或&#x200B;**功能管理員**&#x200B;角色的成員可以核准或拒絕存取要求。

## 可用的角色 {#roles}

指派給您的角色會決定您可在主控台中執行的動作。 如果您不確定要請求哪個角色，請聯絡團隊管理員。

## 在授與存取權之後 {#after-access}

您的請求獲得核准後，請登入Experience Rollouts主控台，並確認您的團隊和應用程式是否可見。 請參閱[登入主控台](log-in-to-the-console.md)以瞭解後續步驟。

## 需要協助嗎? {#need-help}

如果您不確定要加入哪個團隊，或您的專案尚無團隊，請聯絡貴組織的體驗轉出管理員。
