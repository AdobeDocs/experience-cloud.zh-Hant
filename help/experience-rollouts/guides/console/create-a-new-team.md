# 建立新團隊 {#create-a-new-team}

如果您的專案在「體驗轉出」中沒有團隊，則需要先請求團隊，才能開始建立功能標幟。 團隊建立是透過支援要求來處理。

## 開始之前 {#before-you-begin}

確認您的專案尚無團隊。 請洽詢您的產品或工程負責人，或檢閱[團隊及其管理員](teams-and-admins.md)。 如果團隊已經存在，請改為要求加入該團隊 — 請參閱[要求存取權](request-access.md)。

## 提交團隊請求 {#submit-request}

請聯絡Experience Rollouts支援以請求新團隊。 在您的請求中提供下列資訊：

| 欄位 | 說明 |
|---|---|
| **團隊名稱** | 您團隊的唯一名稱。 |
| **目標定位模式** | 您的團隊將使用的目標定位模式： [使用者層級目標定位，或組織和環境層級目標定位](../../getting-started/experience-rollouts-modes.md)。 |
| **環境** | 您需要的環境：預備、生產或兩者。 |
| **團隊管理員** | 將擔任團隊管理員的一或多個人員的電子郵件地址。 |

>[!NOTE]
>
>團隊管理員負責管理成員及核准存取要求。 指定至少兩名管理員，以避免在無法取得一名管理員時產生存取間隙。

## 建立團隊後 {#after-creation}

建立團隊後，管理員就可以開始以自助方式新增成員。 新成員會透過存取管理系統要求存取權，而管理員會核准或拒絕這些要求。

請參閱[要求存取權](request-access.md)，以瞭解成員加入團隊時應該遵循的步驟。
