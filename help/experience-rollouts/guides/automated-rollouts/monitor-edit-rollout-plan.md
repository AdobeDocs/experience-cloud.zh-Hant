# 監視和編輯轉出計畫 {#monitor-edit-rollout-plan}

一旦自動轉出上線，您就可以追蹤其進度並在需要時進行干預。 此頁面說明如何讀取進度指示器、暫停並繼續計畫，以及如何在必要時中止計畫。

本頁假設您已建立並發佈轉出計畫。 如果您尚未設定自動轉出，請參閱[建立自動轉出](create-automated-rollout.md)。

## 檢視轉出進度 {#view-progress}

轉出計劃開始執行後，請開啟功能的&#x200B;**對象**&#x200B;索引標籤以檢視計畫的目前狀態。 計畫會顯示進度指示器，以標籤目前作用中的階段區塊。

使用進度指示器來瞭解下列內容：

* **已完成的區塊** — 指標上方的所有階段和等待區塊都已執行。 這些區塊已鎖定，無法編輯。
* **目前的區塊** — 目前反白的區塊是作用中階段或等待期間。
* **即將到來的區塊** — 指標下的所有階段和等待區塊尚未執行。 仍可編輯這些專案。

## 暫停轉出計畫 {#pause}

若要在轉出進行期間隨時暫停轉出，請在主控台中選取&#x200B;**暫停轉出**。

暫停計畫時：

* 最後一個完成階段區塊的對象仍會維持作用中。 該對象中的使用者會繼續收到該功能。
* 功能標幟、功能群組或跨團隊功能群組保持其目前的開啟或發佈狀態。 它會持續為目前的活躍對象提供此功能。
* 即將推出的階段和等待區塊仍可編輯。

若要繼續暫停的計畫，請選取&#x200B;**繼續轉出**。 計畫會從中斷的地方繼續。

## 中止轉出計畫 {#abort}

若要永久停止轉出計畫，請在主控台中選取&#x200B;**中止轉出**。

計畫中止時：

* 上一個完成階段區塊的對象仍會維持作用中，而功能仍會繼續提供給該對象。
* 功能本身並未關閉 — 如果您要完全停止提供功能，則必須個別中止發行或關閉功能標幟或群組。
* 無法再編輯即將到來的階段和等待區塊。
* 中止的計畫無法繼續。

>[!IMPORTANT]
>
>中止轉出計畫不會自動關閉功能。 如果您要停止向使用者提供功能狀態，請另外對功能狀態執行動作。

## 編輯即將到來的階段區塊 {#edit-upcoming}

計畫進行中（或已暫停）時，您仍可編輯任何尚未執行的階段或等待區塊：

* 調整即將來臨的階段區塊的對象規則或百分比。
* 變更即將到來的等待區塊的持續時間或固定日期。

已完成的區塊已鎖定且無法變更。
