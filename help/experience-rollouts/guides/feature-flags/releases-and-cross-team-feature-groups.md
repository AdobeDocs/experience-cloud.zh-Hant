# 發行和跨團隊功能群組 {#releases-and-cross-team}

發行版本和跨團隊功能群組都支援跨多個團隊和應用程式協調轉出。 正確的選擇取決於您的快取需求、對象目標定位需求，以及您想要對程式有多大的控制權。

## 跨團隊功能群組 {#cross-team-feature-groups}

跨團隊功能群組可讓您從不同團隊擁有的應用程式中捆綁功能標幟，並使用豐富的對象條件來管理這些功能標幟。

主要特性：

* **自助服務** — 不需要任何支援要求即可建立自助服務
* **多樣化對象鎖定目標** — 支援完整的對象條件集（設定檔屬性、內容變數、百分比轉出）
* **您可以建立的數字沒有限制**
* **無快取** — 功能標幟評估需要API呼叫；不支援SDK層級的快取

如需逐步建立指示，請參閱[建立跨團隊功能群組](create-cross-team-feature-group.md)。

## 發行版本 {#releases}

此版本是針對需要SDK層級快取的大型協調啟動所設計。 發行版本使用驗證權杖型對象目標定位。

主要特性：

* **需要支援要求** — 發行並非完全自助式；請連絡Experience Rollouts支援以建立版本
* **SDK快取** — 所有發行版本資料都會快取到本機伺服器SDK中，減少API呼叫
* **有限的對象條件** — 對象規則是根據驗證權杖（國家/地區、電子郵件、使用者ID、百分比等）
* **有限的有效發行版本** — 一次可以存在有限數目的有效發行版本；計畫在三個月內關閉或稽核發行版本

## 比較 {#comparison}

| | 跨團隊功能群組 | 版本 |
|---|---|---|
| 自助服務 | ✓ | 需要支援要求 |
| 豐富的對象條件 | ✓ | 有限 |
| A/B 測試 | ✗ | ✗ |
| SDK快取 | ✗ | ✓ |
| 作用中的限制 | 無限制 | 有限 |

## 建議 {#recommendation}

如果您的使用案例涉及豐富的對象目標定位和自助管理，請使用&#x200B;**跨團隊功能群組**。 如果您的後端服務需要SDK層級的快取，且較簡單的發行對象條件已足夠，請使用&#x200B;**發行版本**。
