# 建立跨團隊功能群組 {#create-cross-team-feature-group}

## 先決條件 {#prerequisites}

在建立跨專案團隊功能群組之前，請確認下列事項：

* 您可以存取主控台 — 請參閱[登入主控台](../console/log-in-to-the-console.md)
* 您屬於某個團隊，您的應用程式已上線
* 您有&#x200B;**功能管理員**&#x200B;角色 — 請參閱[使用者角色](../teams/user-roles.md)
* 您已建立要包含的功能標幟 — 請參閱[建立您的第一個功能標幟](create-your-first-feature-flag.md)

跨團隊功能群組允許將來自具有豐富受眾定位之多個團隊的應用程式的功能標幟捆綁。 不同於發行版本，這是完全自助式且您可以建立的數量沒有限制。 如需比較，請參閱[發行版本和跨團隊功能群組](releases-and-cross-team-feature-groups.md)。

## 步驟1：建立跨團隊功能群組 {#create}

從主控台的Releases區段開始建立程式：

1. 瀏覽至主控台中的&#x200B;**功能與版本>版本**。
2. 選取&#x200B;**新功能群組（跨團隊）**。

## 步驟2：基本詳細資訊 {#basic-details}

提供標題、索引鍵、說明，以及選用的標籤。 設定以下選項:

* **轉出百分比** — 設定有多少對象會收到此功能。
* **轉出型別** — 選擇「手動」或「自動」。 如需自動轉出運作方式的詳細資訊，請參閱[自動轉出](../automated-rollouts/automated-rollout-concept.md)。

>[!NOTE]
>
>跨團隊功能群組不支援A/B測試。 若要執行A/B測試，請使用標準[功能群組](create-a-feature-group.md) （應用程式必須位於相同的群組中）。

## 步驟3：對象 {#audience}

在&#x200B;**對象**&#x200B;標籤上，啟用&#x200B;**對象規則**&#x200B;並設定：

* **區段** — 選取預先定義的對象區段。
* **其他篩選器** — 直接新增設定檔或內容型條件。
* **應用程式** — 從您的團隊或其他團隊中選取一或多個應用程式。

>[!IMPORTANT]
>
>從其他團隊新增應用程式，可授予該團隊的功能管理員將應用程式的功能標幟新增至此跨團隊功能群組的權利。 您無法自行新增其功能標幟。

## 步驟4：功能 {#features}

在&#x200B;**功能**&#x200B;標籤下，您會看到每個包含的應用程式的方塊：

* 對於您自己團隊中的應用程式，您可以直接新增功能標幟。
* 對於來自其他團隊的應用程式，這些方塊會變灰 — 個別團隊的功能管理員必須新增其標幟。

>[!IMPORTANT]
>
>功能標幟一次只能透過一個方法提供。 將標幟新增至跨團隊功能群組會直接移除在標幟上設定的任何對象或轉出百分比。 已存在於另一個版本或功能群組中的旗標將不會顯示。

## 步驟5：排程（選擇性） {#schedule}

您可以排程跨團隊功能群組在未來日期和時間啟用。 如需詳細資訊，請參閱[排程](schedule.md)。

## 管理狀態 {#states}

建立跨團隊功能群組的團隊擁有該群組，並可管理其狀態。 所屬團隊的「功能管理員」成員可使用狀態下拉式清單在狀態之間轉換：

| 狀態 | 說明 |
|---|---|
| **儲存** | 已儲存且未上線。 允許所有變更。 |
| **發佈** | 針對已設定的對象上線。 功能管理員可以繼續更新對象條件和百分比轉出。 |
| **完整轉出** | 可供所有使用者使用。 在此時間點後，無法進一步限制對象。 |
| **基準線** | 功能現在為預設值。 跨團隊功能群組已關閉。 |
| **中止** | 轉出已停止。 沒有使用者會從此群組接收功能。 |

## 常見問題集 {#faqs}

**跨團隊功能群組的數量是否有限制？**
不可以。 不同於發行版本，沒有限制。 不過，當不再需要時，可封存或停用群組。

**發行版本和跨團隊功能群組之間有何差異？**
如需詳細比較，請參閱[發行版本和跨團隊功能群組](releases-and-cross-team-feature-groups.md)。
