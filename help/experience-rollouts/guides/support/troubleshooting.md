# 疑難排解 {#troubleshooting}

體驗轉出包含名為「**功能工作台**」的內建診斷工具，可協助您確切瞭解哪些功能會標示特定使用者接收的內容及原因。 用它來調查意外功能行為的報告，而無需在程式碼中重現問題。

## 存取「功能維護作業」 {#access-workbench}

若要開啟「功能維護作業」，請執行下列步驟：

1. 登入「體驗轉出」主控台。
2. 從頂端功能表瀏覽至&#x200B;**Workbench >一般使用者**。
3. 選取&#x200B;**功能**&#x200B;標籤。

## 輸入使用者身分 {#input-identity}

若要查詢特定使用者的功能旗標評估，請在維護作業表單中輸入下列資訊：

* **ID型別** — 選取您正在使用的識別碼型別。 支援的型別包括電子郵件、GUID和訪客ID。 您也可以直接輸入&#x200B;**發行旗標**，以根據使用者的旗標值來查詢為使用者啟用的發行版本。
* **ID值** — 輸入使用者的識別碼。 如果您選取電子郵件作為ID型別，請注意，多個GUID可以與相同的電子郵件地址建立關聯。 GUID下拉式清單可供您切換相關聯的GUID。
* **驗證來源型別** — 如果適用於您的整合，請選取。
* **內容變數** — 可選擇提供會影響對象規則評估的內容索引鍵值配對（例如，國家/地區、裝置型別）。
* **團隊和應用程式** — 選取您要擷取功能旗標的團隊和應用程式。 預設會預先選取您目前的團隊及其應用程式之一。

填寫表單後，選取&#x200B;**擷取功能**。

## 解讀結果 {#interpret-results}

結果取決於您輸入的ID型別。

### 電子郵件或GUID查閱 {#email-guid-lookup}

當您透過電子郵件或GUID查詢使用者時，結果會出現兩個區段：

**發行旗標**

此段落顯示選取之群組與應用程式的三個清單：

* **已啟用** — 此使用者目前有效的發行版本。
* **已停用** — 存在但未為此使用者啟用的版本。
* **未使用** — 發行未附加發行版本的位元。

與目前選取的專案團隊和應用程式相關的發行版本會出現在每個清單的頂端。

**功能標幟和群組**

本節列出所選團隊和應用程式中目前為使用者啟用的所有功能標幟。 每個專案都會顯示：

* 功能標幟鍵和名稱
* 功能群組（如果旗標屬於一個）
* 變體（如果已設定A/B測試）
* 標幟是否為受眾連結

您也可以選取&#x200B;**顯示要求/回應**，以檢查產生結果的確切API要求與回應。

### 非自我電子郵件或非GUID ID {#non-self-lookup}

當輸入的電子郵件不是您自己的電子郵件，或使用非GUID ID型別時，只會顯示&#x200B;**功能標幟和群組**&#x200B;區段。

### 發行旗標查閱 {#release-flag-lookup}

當您輸入&#x200B;**發行旗標**&#x200B;作為ID型別時，只會顯示&#x200B;**發行旗標**&#x200B;區段，列出與該旗標相關聯的使用者已啟用和停用的發行版本。

## 常見問題 {#common-issues}

下表說明常見問題，以及如何使用維護作業來調查這些問題。

| 症狀 | 檢查內容 |
|---|---|
| 使用者未收到應傳送的功能標幟 | 驗證團隊和應用程式選擇。 檢查使用者的ID是否符合對象規則型別（電子郵件、GUID等）。 確認功能處於&#x200B;**發佈**&#x200B;或&#x200B;**完整轉出**&#x200B;狀態。 |
| 所有使用者都未預期地啟用此功能 | 確認功能未套用對象規則，或百分比已設為100%。 檢查功能是否處於&#x200B;**完整轉出**&#x200B;狀態。 |
| 對象規則不符 | 使用具有內容變數的工作台，以確認在執行階段傳遞的值符合對象規則中已設定的值。 |
| 功能標幟狀態正確，但使用者仍看到舊行為 | 檢查為應用程式設定的TTL。 SDK快取功能資料，並且僅在設定的輪詢間隔重新整理。 |
