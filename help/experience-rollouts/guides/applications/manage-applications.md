# 管理應用程式 {#manage-applications}

Experience轉出中的&#x200B;**應用程式**&#x200B;代表您要使用功能旗標控制的服務或產品。 您必須先將至少一個應用程式上線到主控台，您的團隊才能建立功能旗標。

## 誰可以管理應用程式 {#permissions}

只有具有&#x200B;**管理員**&#x200B;角色的成員才能新增或修改應用程式。 如果您沒有此角色，請聯絡您的團隊管理員。

## 新增應用程式 {#add-application}

若要新增應用程式到您的團隊，請參閱[將應用程式上線](onboard-your-application.md)以取得逐步指示。

## 團隊所有權 {#team-ownership}

每個應用程式都只屬於一個團隊。 只有該團隊的成員可以管理應用程式的功能標幟。

此擁有權模型也會影響您建構特徵群組的方式：

* **在您的團隊內** — 產品發行擁有者可以跨相同團隊擁有的應用程式群組功能標幟。 請參閱[功能群組以控制多個功能](../../concepts/feature-groups-to-control-multiple-features.md)。
* **跨團隊** — 如果您需要從不同團隊擁有的應用程式中將功能標幟分組，請使用跨團隊功能群組，這需要&#x200B;**功能管理員**&#x200B;角色。
