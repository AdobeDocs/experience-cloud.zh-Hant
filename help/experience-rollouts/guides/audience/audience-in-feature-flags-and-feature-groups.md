# 功能標幟和功能群組中的對象 {#audience-overview}

體驗轉出為功能標幟和功能群組提供豐富的受眾目標定位。 您可以結合設定檔屬性、內容變數和對象區段，以精確控制哪些使用者會收到功能。

## 設定檔規則 {#profile-rules}

設定檔規則可讓您根據國家、電子郵件網域、帳戶型別、使用者ID等屬性來鎖定使用者。 您可以使用AND/OR運運算元結合多個屬性，並為複雜的鎖定目標案例建立巢狀邏輯。

若要新增設定檔規則，請移至功能標幟或功能群組的&#x200B;**對象**&#x200B;標籤，並在&#x200B;**設定檔**&#x200B;下新增條件。

您可以將一組對象規則儲存為可重複使用的&#x200B;**對象**，以便用於多個旗標和群組。

如需以內容為基礎的鎖定目標（例如以使用者的使用中語言或使用者端型別鎖定目標），請參閱[在對象規則中使用內容](using-context-in-audience-rules.md)。

## 區段 {#segments}

功能標幟和功能群組支援受眾區段。 區段可讓您：

* 對多個功能標幟和群組使用單一預先定義的對象定義
* 在您的對象目標定位中加入&#x200B;**事件型條件** — 單獨使用設定檔規則是不可能的

若要使用區段，請前往&#x200B;**對象**&#x200B;標籤，然後從清單中選取一或多個區段。 您可以使用AND/OR運運算元結合多個區段，並連同其他設定檔或內容篩選器一併新增。

>[!NOTE]
>
>您需要&#x200B;**測試管理員**&#x200B;角色或更新版本才能建立對象區段。

## 對象電腦 {#audience-calculator}

定義您的對象條件後，選取底部列中的&#x200B;**計算**&#x200B;以取得符合資格的使用者預估人數。 這可協助您在啟動功能之前驗證目標定位的觸及範圍。

>[!NOTE]
>
>當條件中包含內容變數時，對象計算器不會傳回值，因為內容值是在執行階段中決定，而且無法預先預測。
